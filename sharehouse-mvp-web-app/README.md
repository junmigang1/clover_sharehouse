# 우리집 · 셰어하우스 (프로토타입)

셰어하우스 **입주 후 생활**을 위한 커뮤니티 + 관리 앱의 하이파이 인터랙티브 프로토타입입니다.
집을 "구하는" 단계가 아니라, 이미 같이 사는 사람들의 **소통·정산·집안일·중고거래·커뮤니티**에 초점을 맞췄습니다.

> 사용자 인터뷰용으로 만든 클릭 가능한 목업입니다. 실제 결제·AI 호출·서버 연동은 없고, 모든 데이터는 더미입니다.

---

## 실행 방법

```bash
npm install
npm run dev
```

브라우저가 자동으로 열립니다 (기본 `http://localhost:5173`).
데스크탑에서는 가운데 폰 목업으로, 모바일에서는 전체 화면으로 보입니다.

빌드하려면:

```bash
npm run build      # 타입체크 + 프로덕션 빌드 → dist/
npm run preview    # 빌드 결과 미리보기
```

### Figma Make에 올리는 경우
`src/` 폴더의 컴포넌트/페이지를 그대로 붙여넣어 사용할 수 있습니다.
디자인 토큰은 `src/index.css` 상단 `:root`에 모여 있어 색/폰트만 바꿔도 전체 톤이 바뀝니다.

## 인터뷰 시나리오 (핵심 플로우)

1. **홈** — 오늘의 청소 담당, 안 낸 지출, 최근 공지, 다가오는 일정, 내 집안일을 한눈에
2. **공지 확인** — 홈 → 공지 카드 → 상세
3. **공유 지출 정산** — 홈/전체보기 → 항목 → `정산하기` → 결제수단 선택 → 완료 (목업)
4. **커뮤니티** — 카테고리 필터, 투표 참여, 좋아요/댓글 작성
5. **마켓** — 카테고리별 중고거래, 상품 상세 → 채팅 (목업)
6. **AI 도우미** — 공지 작성 / 청소 스케줄 / 하우스룰 / 주간요약 생성 (미리 준비된 예시 결과)
7. **MY** — 프로필, 하우스 정보, 멤버, 청소 로테이션, 알림 설정

거의 모든 화면이 클릭으로 이어지므로, 인터뷰 대상자에게 직접 만지게 하며 반응을 관찰하기 좋습니다.

## 폴더 구조

```
src/
├─ main.tsx              진입점
├─ App.tsx               Provider + 프레임 + 라우터 + 하단탭 (얇게 유지)
├─ index.css             디자인 토큰 · 공용 스타일 (여기서 색/폰트 관리)
├─ types.ts              공용 타입
├─ hooks/
│   └─ useNavigation.tsx  스택 기반 화면 전환 (라우터 라이브러리 없이)
├─ components/           재사용 UI (Icon, Avatar, Card, TopBar, BottomNav, Router …)
├─ pages/                화면별 컴포넌트 (Home, Expenses, Community, Marketplace, AI, My …)
└─ data/                 더미 데이터 (멤버·공지·지출·커뮤니티·마켓·AI)
```

새 화면 추가는 ① `pages/`에 컴포넌트 만들고 ② `hooks/useNavigation.tsx`의 `RouteKey`에 키 추가 ③ `components/Router.tsx`에 연결하면 됩니다.

## 디자인 노트

- **팔레트**: 세이지 그린(집·주방·식물의 차분함) 중심. 흔한 SaaS 블루/테라코타를 의도적으로 피함.
- **폰트**: Pretendard (한국어 UI에 최적화, iOS 느낌). CDN 로드, 오프라인 시 시스템 폰트로 대체.
- **모션**: 화면 진입 페이드업 + 눌림 피드백만. `prefers-reduced-motion` 존중.

## 기술 스택

React 18 · TypeScript · Vite. 외부 UI 라이브러리 없이 순수 CSS + 인라인 SVG 아이콘으로 구성해 셋업이 가볍습니다.
